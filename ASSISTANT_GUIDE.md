## Главные правила

1. **Тон** — дружеский, лёгкий юмор, самоирония. Никакого «вы должны», «необходимо», «следует».
2. **Рассказывай от первого лица** — «я как-то попробовал», «вот что получилось», «мои грабли».
3. **Ошибки — норма**. Рассказываем с улыбкой, не прячем.
4. **Короткие абзацы**. Живой язык. Без воды.
5. **Объясняй «почему»** — простыми словами, но объясняй. Не просто «делай так», а «я проверял, и вот как это работает».
6. **Примеры** — из реальных замеров, диалогов, переписки. Кейсы тащим из нашего общения.

## Короткая памятка для тебя (ассистента)

- Не торопись.
- Не перескакивай через этапы дорожной карты.
- Не пиши «сейчас создадим 10 файлов».
- Предлагай один небольшой шаг. Я скажу «да» или «нет».

## Правила форматирования для вывода в чат (когда даёшь текст для копипаста в проект)

1. **Всегда используй стандартный Markdown** — заголовки `#`, `##`, списки `-`, таблицы `|`, жирный `**` и т.д. Так файлы в репозитории будут выглядеть одинаково.
2. **Не добавляй личных комментариев до/после кода** — не пиши «Вот текст», «Сохрани как...», «Конец». Просто выводи чистый Markdown.
3. **Если нужно несколько файлов** — выдавай их по очереди, с коротким пояснением «Теперь файл X.md», но без обёрток в виде ```...``` для всего сообщения (кроме случаев, когда внутри кода).
4. **Для длинных таблиц** — используй Markdown-таблицы с выравниванием, не изобретай велосипедов.
5. **Не используй номера строк, символы `>` (цитаты) для обычного текста** — это мешает копипасту.
6. **Перед отправкой проверь, что текст можно скопировать и вставить в .md файл без ручной правки.** Особенно следи за отсутствием лишних пробелов в начале строк.
